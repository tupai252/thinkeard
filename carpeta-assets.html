Como comenzar

Tutorial: Abrir y ejecutar un proyecto en Unity

Requisitos

  • Tener Unity Hub instalado (descargar aquí).
  • Tener instalada una versión de Unity compatible con el proyecto.

Pasos

1. Abrir Unity Hub

Ejecuta Unity Hub desde tu computadora.

2. Agregar el proyecto

  • En Unity Hub, ve a la pestaña "Projects".
  • Haz clic en el botón "Open" o "Add".
  • Navega hasta la carpeta donde está guardado el proyecto (donde ves carpetas como Assets/ y ProjectSettings/).
  • Selecciona la carpeta del proyecto y confirma.

3. Seleccionar la versión correcta de Unity

  • Si ves un aviso de "Incompatible version", instala la versión recomendada o abre con una versión similar (puede haber avisos menores).

4. Abrir el proyecto

  • Haz clic en el proyecto para abrirlo.
  • La primera vez puede tardar mientras se importan archivos.

5. Ejecutar el proyecto

  • Abre la escena principal (generalmente en Assets/Scenes/).
  • Haz doble clic en la escena que quieras correr.
  • Presiona el botón "Play" (ícono ▶️ en la parte superior).
  • ¡Listo! El proyecto debería ejecutarse en el editor.

Carpeta Raíz

Información de carpetas principales

  • Assets: Carpeta principal que contiene todos los recursos editables del juego , como codigo, escenas, texturas, y sonidos.
  • Packages: Contiene los paquetes utilizados por el proyecto , gestionados por Unity Package Manager. Incluye dependencias como Unity UI, TextMeshPro, etc.
  • ProjectSettings: Almacena las configuraciones globales del proyecto, como ajustes de renderizado, física, y entrada.
  • Assembly-CSharp (54 KB): Archivo de proyecto de C# que organiza los scripts del juego. Los programadores pueden abrir este archivo en un editor de código para trabajar en los scripts.
  • juego.sln (1 KB): Archivo de solución de Visual Studio. Es el punto de entrada para editar los scripts en un editor como Visual Studio o VS Code.

Scripts Importantes

Esta carpeta tiene recursos editables por codigo del juego del juego. Se mostrará por subcarpetas.

  • Codigo: Contiene los scripts en C# del juego. Los scripts encontrados incluyen:
    • ControlColisiones.cs: Maneja la lógica de colisiones entre objetos.
    • Controles.cs: Gestiona las entradas del jugador (movimiento, acciones, etc.).
    • Creditos.cs: Controla la pantalla de créditos del juego.
    • EliminarFIGURA.cs: Maneja la eliminación de figuras u objetos.
    • jugar.cs: Gestiona la funcionalidad para iniciar el juego .
    • MoverCamara.cs: Controla el movimiento de la cámara .
    • movimiento_figura.cs: Maneja el movimiento de una figura u objeto específico.
    • MOVIMIENTOS_FIGURAS.cs: Gestiona el movimiento de múltiples figuras u objetos.
    • MusicManager.cs: Controla la música del juego.
    • Opciones.cs: Maneja la pantalla de opciones o configuración.
    • Seleccionarfigura.cs: Permite seleccionar una figura u objeto .
    • SpawnManager.cs: Gestiona la generación de objetos .
    • VolumeControl.cs: Controla el volumen del juego (no funciona del todo, requiere arreglo). Nota: Consulte el rastreador de errores para más detalles.
    • VOLVER.cs: Maneja la funcionalidad de "volver" (regresar al menú principal).
  • Scenes: Contiene las escenas del juego, como Menu.unity y Opciones.unity
  • Textures: Almacena las texturas utilizadas en el juego, como imágenes de objetos y escenarios.
  • Prefabs: Objetos prefabricados que se pueden reutilizar en las escenas.

Nota: Por cierto, si buscan archivos específicos tendran que buscar ya que este es resumido, exploren la carpeta Assets en el editor de Unity o en el explorador de archivos.
